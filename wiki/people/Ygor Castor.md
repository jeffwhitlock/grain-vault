# Ygor Castor

Backend engineer at [[Grain]]. Works on AI infrastructure, entity extraction, and the [[Knowledge Graph System]] / entity memory project.

## Update 2026-05-18 from [[2026-05-18_check-in-on-entity-md-docs-for-the-mcp]]

Led the technical discussion on implementing entity markdown docs for the [[Grain MCP]]. Proposed starting simple: a single markdown file per company entity, updated after each meeting using condensed outputs (notes + action items, not full transcripts).

Cautioned against over-engineering: "The memory is a kind of unsolved problem right now. There are several different approaches from several different people, from graphs to simple embeds to a lot of stuff."

Aligned with [[Jeff Whitlock]] on starting with company-level entities, bootstrapping by processing the last few meetings for a company when a new meeting triggers creation, and avoiding full knowledge graph complexity.
